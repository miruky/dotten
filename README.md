**ブラウザでドット絵を描き、矩形にまとめた軽いSVGとして書き出すピクセルアートエディタ。**

公開ページ: https://miruky.github.io/dotten/

## 概要

dottenは小さなドット絵(8×8・16×16・32×32)を描いてSVGで持ち出すためのエディタである。ペン・消しゴム・塗りつぶし・スポイトの4つの道具と編集できるパレットで描き、左右対称描画・取り消し・やり直しで気軽に試行錯誤できる。書き出すSVGは塗りマスを矩形にまとめた最適化済みの出力で、サイドパネルにそのプレビューと矩形数が常に表示される。SVGのほかPNGでも書き出せ、描いた絵はURLに畳んで共有できる。

描きかけの絵と道具の設定はブラウザのlocalStorageに保存され、サーバーには何も送らない。

### なぜ作ったのか

ファビコンやUIの小さなアイコンをドット絵で作るとき、本格的なピクセルアートツールでは大げさで、書き出しがPNGになるのも困る。スケーラブルに使い回すにはSVGがほしいが、1マス=1`rect`の素朴な変換は16×16でも最大256個の`rect`になってしまう。そこで「すぐ描けて、軽いSVGが出てくる」の2点に絞り、塗りを矩形へ貪欲に結合して`rect`数を抑える書き出しを中心に据えた。

## アーキテクチャ

![構成図](docs/architecture.svg)

UI層はフレームワークなしのTypeScriptで、盤面のDOMは描画のたびに作り直さず、塗ったマスの背景色だけを差し替える(ドラッグ中の描画を軽くするため)。盤面の操作・SVG生成・共有文字列の符号化・設定の検証はDOMに依存しない純粋なモジュールに分け、そのまま単体テストできる。

## 使い方

### 描く

道具はペン・消しゴム・塗りつぶし・スポイトの4つ。マウスでもタッチでも、なぞった軌跡のマスが塗られる。塗りつぶしは同じ色の連結した領域(上下左右)をまとめて塗り替える。スポイトは盤面の色を拾ってペンに持ち替える。**対称**を入れると左右対称に同時に塗れる。1ストロークが取り消しの1単位で、最大60手まで戻れる。

パレットの色は1から0のキーで選べる。自由な色は色選択から作り、**追加**でパレットへ常駐させられる。直前に使った色は「最近の色」に並ぶ。

### キーボード

| キー                  | 動作                                    |
| :-------------------- | :-------------------------------------- |
| `P` / `E` / `F` / `I` | ペン / 消しゴム / 塗りつぶし / スポイト |
| `M` / `G`             | 左右対称 / 格子の表示                   |
| `1`–`9` `0`           | パレットの色を選ぶ                      |
| `[` / `]`             | 盤面を小さく / 大きく                   |
| `T`                   | テーマ切替                              |
| `Ctrl`/`Cmd`+`Z`      | 取り消し(`Shift`併用でやり直し)         |

### 盤面の大きさ

8×8・16×16・32×32を切り替えられる。広げたときは左上を基準に既存の絵が保たれ、狭めたときは右下が切り落とされる(この操作も取り消せる)。

### 書き出し

「SVGをコピー」「SVGを保存」でSVGを、「PNGを保存」で倍率(8〜32倍)を選んでPNGを書き出す。SVGの出力は次の形になる。

```svg
<svg xmlns="http://www.w3.org/2000/svg" viewBox="0 0 16 16" shape-rendering="crispEdges" role="img" aria-label="dot art">
  <title>dot art</title>
  <rect x="8" y="9" width="1" height="4" fill="#4a7c3a"/>
  <rect x="5" y="13" width="6" height="1" fill="#8c7a66"/>
</svg>
```

- まとめ方は**矩形**と**行**の2通り。矩形は同色のマスを縦横に揃えて貪欲に結合し`rect`数を最小化する。行は同じ行の連続だけをまとめる。プレビューの下に現在の`rect`数と塗りマス数を表示する。
- `viewBox`は1マス=1単位。`width`/`height`を付けないので、使う側で自由な大きさに伸縮できる。
- `shape-rendering="crispEdges"`で拡大時のにじみを防ぐ。

### 共有

「共有リンクをコピー」で、盤面をパレット索引のランレングスにしてbase64urlでURLのハッシュに畳んだリンクを作る。リンクを開くとその絵が読み込まれる(サーバーは介さない)。

### 制約

- 持てる絵は1枚だけ。複数の作品を切り替える機能はない。
- 矩形の結合は行ランの位置と幅が揃った場合に縦へ伸ばす貪欲法で、任意形状に対する最小分割ではない。
- レイヤー・アニメーションには対応しない。

## プロジェクト構成

- `index.html` — エントリポイント。描画前にテーマを確定する起動スクリプトを含む
- `src/main.ts` — 起動。共有リンクの読み込み、ストアと設定の初期化、初回の見本データ投入
- `src/app.ts` — 道具・パレット・取り消し・書き出し・ショートカットの画面とイベント処理
- `src/icons.ts` — 線画SVGアイコン
- `src/style.css` — デザイントークンとスタイル(罫線基調・市松の透明表現・ライト/ダーク)
- `src/lib/grid.ts` — 盤面の型・塗りつぶし・大きさ変更・対称点・永続化
- `src/lib/svg.ts` — 行ランと矩形結合をまとめたSVG書き出し
- `src/lib/encode.ts` — 共有用のbase64url符号化と復号
- `src/lib/palette.ts` — 色入力の正規化と最近使った色
- `src/lib/theme.ts` — テーマ選択値の解決
- `src/lib/settings.ts` — 道具設定の永続化と検証
- `src/lib/seed.ts` — 初回起動時の見本のドット絵
- `docs/architecture.svg` — 構成図
- `.github/workflows/` — CI(lint・テスト・ビルド)とPagesデプロイ

## はじめ方

### 前提条件

- Node.js 22以上

### セットアップ

```bash
git clone https://github.com/miruky/dotten.git
cd dotten
npm install
npm run dev
```

### テストの実行

```bash
npm test
```

### Lintの実行

```bash
npm run lint
```

### ビルド

```bash
npm run build
```

GitHub Pagesではリポジトリ名のサブパスで配信されるため、デプロイ時は環境変数 `DOTTEN_BASE=/dotten/` でViteの `base` を切り替える(`.github/workflows/deploy.yml` 参照)。

## 設計方針

- **出力の質を機能にする** — 書き出すSVGの軽さと素性の良さ(viewBox・crispEdges・矩形数)をプレビューで常に見せる。矩形結合と行結合を切り替えて結果を比べられる。エディタの主役は編集画面ではなく出力だと考えた。
- **描き味を落とさないDOM更新** — ドラッグ中は塗ったマスのスタイルだけを書き換え、画面全体の再描画をしない。1ストローク=取り消し1単位も、この境界で確定する。
- **盤面とデータは純粋な関数** — 塗り・塗りつぶし・大きさ変更・対称・符号化・設定検証は配列や文字列に対する純粋な操作で、エッジケース(範囲外・同色塗りつぶし・縮小の切り落とし・255マス超のラン・壊れた保存値)をテストで固定している。
- **入力は寛容に、保存は厳密に** — 色入力は3桁・6桁・前置きの`#`有無を受けつつ`#rrggbb`へ揃え、保存データの復元は大きさ・色形式・設定値を検証して、壊れていれば既定から始め直す。
- **罫線と余白で構成する** — 厚い枠と影を重ねず、1pxの罫線と余白、明確なタイポ階層で道具を並べる。テーマは`data-theme`で切り替え、描画前に確定してちらつきを防ぐ。動きは控えめにし、`prefers-reduced-motion`で止められる。
